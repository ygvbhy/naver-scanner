## 0. 프로젝트 개요

- 프로젝트 목적:

  - 도·소매 납품가 및 권장 소비자가(RSP)를 결정하기 전에
    네이버 쇼핑 기준 오픈마켓 최저가를 사전에 확인하여
    가격 역전, 유통사 컴플레인, 추가 할인 요구 등의 리스크를 줄이기 위함

- 주요 사용자:

  - 내부 MD
  - 영업 / 유통 담당자
  - 가격 정책 결정자

- 해결하려는 문제:
  - 현재 오픈마켓 최저가를 사전에 파악하기 어려움
  - 가격 책정 이후 발생하는 역마진 및 클레임 리스크
  - 근거 없는 가격 결정으로 인한 내부 커뮤니케이션 비용 증가

## 1. 기술 스택 고정 (변경 시 명시적 논의 필요)

- Framework:

  - Next.js (App Router)
  - React 보안 취약점이 해결된 최신 안정 버전 사용

- Language:

  - TypeScript

- Styling:

  - Tailwind CSS

- State Management:

  - 기본은 React 로컬 상태
  - 전역 상태 필요 시 Zustand 사용

- Server State:

  - 필요 시 TanStack Query 사용
  - 서버 상태와 UI 상태 분리 원칙 유지

- Performance:
  - React Compiler 사용
  - useMemo / useCallback은 문제 발생 시에만 적용
